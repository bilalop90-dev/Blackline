How this works

  1. Your PDF is opened with JavaScript's FileReader API and rendered with Mozilla's pdf.js — entirely inside this browser tab. No upload, no server, no exceptions.

  2. You draw redaction boxes over anything sensitive. The boxes are tracked per page in your browser's memory only.

  3. On export, every page is re-rendered as a flat image with your boxes burned in as solid black, then rebuilt into a fresh PDF with pdf-lib. The original text layer, fonts, and metadata are not copied over — the content under a box isn't hidden, it no longer exists.

  4. The redacted PDF is assembled in memory and handed to your browser's download manager. Try copy-pasting over a black box in the result — there's nothing there to copy.

You can verify this yourself: open DevTools → Network tab → load a PDF, redact, and export. You will see zero outbound requests carrying your file — the only network traffic is this page's own static assets.
